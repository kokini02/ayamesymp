📄 各ファイルの役割と詳細解説
このシステムは、**「情報を発信するページ（閲覧用）」と「情報を更新するページ（管理用）」**の2つの役割に分かれています。

1. index.html（サイトの顔・看板）
役割: バンドの第一印象を決めるトップページです。

表示内容: 最新の演奏会情報、ブログの最新3件、楽器別のざっくりとした募集状況、フォトギャラリーを表示します。

使いかた: ユーザーがアクセスした際、裏側でFirebaseに「最新の情報をちょうだい」とリクエストを送り、自動的に内容を書き換えます。自分でこのファイルを直接編集する必要はありません。

2. admin.html（コントロールパネル）
役割: サイトのすべての情報を操る「司令塔」です。

機能: 演奏会の日時、ブログ記事の投稿、パートごとの細かい人数調整、写真の追加など、サイト上のほぼすべてのデータをここで編集します。

使いかた: 団の運営担当者がこのページにログインし、フォームに入力して「更新」ボタンを押します。ボタンを押した瞬間にクラウド上のデータが書き換わり、公開サイトに反映されます。

3. members.html（在籍人数の詳細名簿）
役割: パートごとの正確な人数（活動中・休団中）と、詳細な募集条件を提示するページです。

特徴: 単なる表ではなく、JavaScriptが「活動人数 ＋ 休団人数 ＝ 総員」を自動で計算して合計欄に表示します。

使いかた: 入団希望者が「自分のパートは今何人くらいいるのかな？」と確認するために参照します。

4. blog.html（活動日誌アーカイブ）
役割: これまでに投稿されたすべてのブログ記事を時系列（新しい順）で表示するページです。

特徴: 文字数が多い記事でも読みやすいよう、専用のレイアウトになっています。

使いかた: 練習の様子や本番の記録など、バンドの雰囲気を知りたい人向けの読み物コンテンツとして機能します。

5. recruit.html（入団希望者向け案内）
役割: 入団を検討している人に対して、募集状況と活動ルール（団費や練習場所）を伝えるページです。

使いかた: admin.html で入力された募集人数や「急募」などのステータスを読み取り、自動で色分け（急募なら赤色など）して表示します。

6. firebase-config.js（接続の鍵）
役割: サイト全体をFirebaseというクラウドデータベースに繋ぐための「中継地点」です。

使いかた: このファイルにあなたのFirebaseプロジェクト専用のIDやキーを書き込んでおくことで、他のすべてのHTMLが迷わずデータベースにアクセスできるようになります。

🔄 運用の流れ（使いかた）
ログイン: login.html から管理画面へ入ります。

編集: admin.html で、例えば「次の演奏会が決まった」なら、そのタイトルと場所を入力して保存します。

自動反映: 保存した瞬間に、Firebaseを通じて index.html の「Next Concert」の欄が新しい情報に書き換わります。

メンテナンス: 人数に変動があったら、admin.html の表の数字をいじるだけで、members.html と recruit.html の両方が同時に最新の状態になります。
